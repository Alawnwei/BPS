  ⚠️需要注意的问题

  1. 核心逻辑：赌徒谬误（Gambler's Fallacy）

  几乎所有策略都建立在一个错误的假设上："过去的结果影响未来的概率"

  长龙回头不过3：6连庄断了 →赌闲不会3连
  →但每一手庄/闲的概率始终是 ~50%（去和）
  →之前的6连庄不影响下一手

  这是百家乐中最常见的认知误区。单跳不是"高概率"，只是因为交替序列的种类比连庄连闲多，视觉上更常见。

  2. 负追的数学风险

  书中多个策略使用负追（输了加注），这是数学上危险的做法：

  第1枪输 100 →第2枪输 200 →第3枪输 400
  3枪全输 = 亏损 700
  要一次赢回来需要连中多次

  百家乐不存在"必中"的负追，长龙可以比你的负追枪数更长。

  3. 选择性展示

  书中展示了大量"命中案例"，但没有展示完整的统计。随机抽取几段路来证明胜率，是典型的幸存者偏差——这是作者自己在序言里批判
  过的问题。

  4. 过度复杂的规则体系

  每个策略都有大量例外条件（双联龙慎用、金手指避开、比例悬殊降注码…），说明：
  - 策略本身没有稳定的统计优势
  - 需要大量主观判断来"筛选"适用场景
  - 不同玩家的判断会导致截然不同的结果

  目前只有3-4个策略有"数据验证"标注，其他都在用"中""高""极高"做模糊化表述。要么全部标注回测胜率，要么把信心等级改为"形态强度"而非"预测准确率"。
  2. 引入"长龙不过3"的数学期望计算
  对于负追策略，不仅要看胜率，还要看赔率结构。如果胜率只有33.3%但负追3枪，数学期望是负的 — 系统应该计算并展示这个期望值，不应该是纯语言描述。
  3. 三路分析目前只是展示
  大眼仔/小路/甲由路已经计算了（2604行附近），但并未集成到推荐引擎中。目前三路只是一个状态摘要标签，没有影响策略输出。要么移除，要么真正用起来。
  4. 增加会话统计和复盘导出功能
  这是职业玩家最需要的东西 — 能够导出自己的下注记录、按策略统计盈亏、分析自己的行为偏差。目前系统会记录sessionStats但没有导出功能。
  5. 凯利公式的胜率输入需要校准
  calculateKellyBet 基于策略历史胜率计算，但样本量不足时（<5次）会有极不稳定的输出。应该用贝叶斯收缩（向先验50%收缩）来处理小样本情况。

  ---
  当前系统的滚动窗口（15手）的问题

  系统用15手的滚动窗口来做降级判断，它的统计功效几乎为零：

  15手，胜率60%（赢了9手）
  95%置信区间 = 60% ± 1.96 × √(0.6×0.4/15)
              = 60% ± 24.8%
              = [35.2%, 84.8%]

  35%到85%——这个区间跨越了从"极差"到"极好"的全部范围，完全无法区分一个策略是否有效。

  15手只适合检测极端异常（比如15手全输），但无法判断一个策略优不优秀。
